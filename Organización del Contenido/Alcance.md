## **CONTENIDO INDEXADO Y BUSCABLE:**

### **Nivel 1: Contenido PRIMARIO (Máxima Relevancia)**

#### **1. Nombres y Modelos de Productos**
- **Indexado:** Nombres completos de productos
- **Ejemplos:** "Taladro Percutor Bosch GBH 2-28", "iPhone 17 Pro Max", "Sofá Seccional Moderno"
- **Prioridad:** ALTA - Es lo que los usuarios buscan primero

#### **2. Marcas de Productos**
- **Indexado:** Todas las marcas mencionadas en cada categoría
- **Ejemplos:** Bosch, Apple, Samsung, LG, Stanley
- **Prioridad:** ALTA - Los usuarios suelen buscar por marca

#### **3. Categorías de Productos**
- **Indexado:** Categorías principales y subcategorías
- **Ejemplos:** "Herramientas eléctricas", "Smartphones", "Muebles de sala"
- **Prioridad:** ALTA - Filtrado fundamental

### **Nivel 2: Contenido SECUNDARIO (Alta Relevancia)**

#### **4. Características Técnicas Específicas**
- **Indexado:** Especificaciones técnicas claves
- **Por categoría:**
  - **Herramientas:** Potencia (850W), dimensiones (185mm)
  - **Tecnología:** Procesadores (M3 Max), RAM (32GB), almacenamiento (1TB)
  - **Muebles:** Materiales (madera, tela), tamaños (king size)
  - **Electrodomésticos:** Eficiencia (A+++), capacidad (18kg)
- **Prioridad:** MEDIA-ALTA - Para usuarios técnicos o específicos

#### **5. Estilos y Diseños (Muebles)**
- **Indexado:** Estilos de diseño específicos
- **Ejemplos:** "Moderno", "Escandinavo", "Industrial", "Minimalista"
- **Prioridad:** MEDIA-ALTA - Esencial para búsqueda de muebles

### **Nivel 3: Contenido TERCIARIO (Relevancia Específica)**

#### **6. Atributos Especiales**
- **Indexado:** A través de data attributes y filtros especiales
- **Ejemplos:**
  - "Profesional" (para herramientas)
  - "Ecológico" (para muebles)
  - "Novedades 2024" (todas las categorías)
  - "Gaming" (para tecnología)
- **Prioridad:** MEDIA - Para búsquedas con requisitos específicos

#### **7. Descripciones de Productos**
- **Indexado:** Según se menciona en "Búsqueda en nombres, descripciones, marcas, modelos..."
- **Prioridad:** MEDIA-BAJA - Contenido complementario

## **CONTENIDO NO INDEXADO :**

### **Lo que NO está incluido en la búsqueda:**

1. **Servicios ofrecidos** 
2. **Secciones informativas** (Nosotros, Servicios)
3. **Contenido textual principal** de páginas estáticas (sobre nosotros, contacto, políticas)
4. **Precios** (solo se usan para ordenar, no para buscar)
5. **Stock/disponibilidad**
6. **Reseñas o calificaciones de productos**
7. **Contenido multimedia** (texto de imágenes, videos)



## **PRIORIZACIÓN POR RELEVANCIA PARA EL USUARIO:**

### **Nivel 1: Esencial para la conversión (COMPRA)**
1. **Nombres de productos específicos** - Usuario sabe exactamente lo que quiere
2. **Marcas** - Usuario es leal a una marca o busca calidad reconocida
3. **Categorías principales** - Usuario está explorando o tiene una necesidad general

### **Nivel 2: Para decisiones informadas**
4. **Características técnicas clave** - Usuario compara especificaciones
5. **Estilos de diseño** - Usuario busca coherencia estética (especialmente muebles)
6. **Filtros especiales** - Usuario tiene requisitos específicos (profesional, ecológico, novedad)

### **Nivel 3: Para refinamiento**
7. **Descripciones generales** - Usuario busca información adicional o keywords secundarias



## **CONCLUSIÓN:**

**El sistema actual indexa principalmente:**
- **Productos** (nombres, modelos)
- **Marcas**
- **Categorías**
- **Características técnicas específicas por categoría**
- **Atributos especiales via data attributes**

**Pero NO indexa:**
- **Contenido informativo** (servicios, secciones de ayuda, blog)
- **Contenido institucional** (páginas estáticas)
- **Contenido de soporte al usuario**

**La priorización actual está bien alineada con las necesidades de compra**, pero **sería recomendable extender el indexado a**:
1. **Servicios** (garantías, instalación, mantenimiento)
2. **Secciones informativas** relevantes (guías de compra, comparativas)
3. **FAQ y contenido de ayuda** relacionado con productos
4. **Artículos del blog** que mencionen productos específicos

**Resumen:** El sistema está optimizado para **búsqueda transaccional** (encontrar productos para comprar) pero no para **búsqueda informativa** (encontrar servicios, información, soporte).
